# mcp/seq-otel

Standalone MCP server container that connects AI agents to an existing Datalust Seq instance.

## What This Image Does

- Exposes a Model Context Protocol (MCP) server over stdio.
- Connects to Seq using `SEQ_URL` and `SEQ_API_KEY`.
- Supports starter tools and full Seq API route coverage through generated MCP tools.

## Required Environment Variables

- `SEQ_URL` - Seq base URL (recommended) or API URL
- `SEQ_API_KEY` - Seq API key

Examples of supported URL formats:

- `http://host.docker.internal:10150`
- `http://host.docker.internal:10150/api`
- `https://seq.example.com`
- `https://seq.example.com/api`

## Quick Run

```bash
docker run --rm -i \
  -e SEQ_URL=http://host.docker.internal:10150 \
  -e SEQ_API_KEY=your-key \
  mcp/seq-otel:local
```

## MCP Client Example

```json
{
  "mcpServers": {
    "seq-otel": {
      "type": "stdio",
      "command": "docker",
      "args": [
        "run",
        "--rm",
        "-i",
        "-e",
        "SEQ_URL=https://seq.example.com",
        "-e",
        "SEQ_API_KEY=<YOUR_SEQ_API_KEY>",
        "mcp/seq-otel:latest"
      ]
    }
  }
}
```

## Permissions

For read/query scenarios, use a least-privilege API key with `Read` permission.

Additional permissions (`Write`, `Project`, `Organization`, `System`, `Ingest`) are only required when invoking endpoints that demand them.
